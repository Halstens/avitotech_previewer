## **Сервис назначения ревьюеров для Pull Request’ов (Avito)**

Cервис, который назначает ревьюеров на PR из команды автора, позволяет выполнять переназначение ревьюверов и получать список PR’ов, назначенных конкретному пользователю, а также управлять командами и активностью пользователей.


#### Использование приложения
1. Клонируете репозиторий командой `git clone <ссылка_на_проект>`
2. Запуск в Bash с помощью Makefile или docker-compose
   - make docker-up \ docker-compose up --build
3. Остановка сервисов
   - make docker-down -> make clean \ docker-compose down -v
4. Сервер развернется сразу на порту 8080
5. Использование по списке реализованных эндпоинтов

Для удобства, расписал список эндпоинтов.

#### API Эндпоинты

1. POST /team/add - Создать команду с участниками
2. GET /team/get - Получить информацию о команде
3. POST /users/setIsActive - Установить флаг активности пользователя
4. GET /users/getReview - Получить PR пользователя как ревьюера
5. POST /pullRequest/create - Создать PR с автоматическим назначением ревьюеров
6. POST /pullRequest/merge - Пометить PR как мердженный
7. POST /pullRequest/reassign - Переназначить ревьюера
8. GET /health - Проверка здоровья сервиса
9. GET /stats - Статистика по PR и назначениям
10. POST /team/bulkDeactivate - Массовая деактивация пользователей команды

["Тестовый" сценарий для понимания использования API](./TESTING.md)
